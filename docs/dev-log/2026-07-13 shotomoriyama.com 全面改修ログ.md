# 2026-07-13 shotomoriyama.com 全面改修ログ

awwwards受賞レベルを目指した全面改修。3本の並行コードレビューから始めて、フェーズ1〜5・全16コミットを **PR #4 として main にマージ、本番反映まで完了**。

## 経緯と方針

- 「作り直し」ではなく「ブランチ＋Vercelプレビューで段階改修」を選択（土台のNext.js 16 + React 19は健全、問題は実装レイヤーに集中していたため）
- 演出は「上品に控えめ」の方針。アニメーションシステムはGSAPを足さずFramer Motion＋Lenisの1系統に統一
- 毎ビルド後に「本番ビルド → 全15ルートをヘッドレスChromium実走 → エラーゼロ確認 → スクリーンショット目視」で検証

## レビューで発覚した重大問題（すべて修正済み）

1. **本番トップページがクラッシュしていた** — レンダー中の `Math.random()` によるSSR/クライアント不一致（React #418）。見出しはSSRで実テキスト描画に変更し、スクランブルはマウント後開始に
2. **`dark:` がテーマトグルに連動していなかった** — Tailwind v4では `tailwind.config.ts` が読み込まれず `darkMode: 'class'` が無視されていた。`@custom-variant dark` で根治（ロゴ切替・ヒーローグラデ等すべて対象だった）
3. **404リンク11件** — サポート記事一覧15件中9件（本文がある6記事のみ公開に変更）＋フッターの `/privacy`・`/terms`（ページ新規作成）
4. **コンタクトAPIが無防備** — zodバリデーション・ハニーポット・IP毎レート制限・エラー内部情報の遮断を追加
5. **SEOメタデータ皆無** — 全サブページに個別メタデータ、sitemapへ動的ルート追加、canonical、JSON-LD、ドメインを `www.shotomoriyama.com` に統一
6. **worksページから個人プロジェクト詳細に到達不能** — カード全体が外部リンクだった。Visit Project / View Details の2導線に再構成

## カーソル差別化の検討メモ

「リング＋拡大」はAI提案が均質化していて審査員にも既視感がある、という問題意識から5案を検討：
A) 波紋・環境反応 / B) ブループリント・レンズ / C) 速度変形ドット / D) 文字残像トレイル / E) コンテキストラベル。
→ **C+Aを採用、Bも実装**。レンズは「綺麗なデザインの裏に丁寧な構造がある」という自分の売りをそのまま体験化した仕掛け。
